FAQ

Short answers for setup, hosting, permissions, persistence, and why NanoBot behaves the way it does.

SetupSelf-hosting
Does NanoBot support slash commands and prefix commands?

Yes. NanoBot commands work as slash commands, prefix commands, and @mention unless noted otherwise.

Which Discord intents are required?

Server Members Intent and Message Content Intent. Prefix commands and several moderation commands fail without them.

Do I need a cloud database?

No. NanoBot uses SQLite as portable local storage with zero cloud dependency.

Does reminder and moderation state survive restarts?

Yes features explicitly timed bans, reminders, recurring reminders, and role panels all persist across restarts via stored state.

Why are admin commands prefix-only?

Slash admin commands would appear in every user's slash menu, so owner admin controls stay prefix-only by design.

Do I need Groq key for whole bot?

No. Groq key is optional and only needed for /eli5 plus daily WYR generation described in README and cogs/eli5.py.

Can I reload config without restart?

Yes. README and admin cog both document reloadconfig and config set/get flows.

Where do logs go?

Logs are written to logs/nanobot.log. docker-compose.yml also mounts host logs directory into container.

Can older JSON installs be migrated?

Yes. Use the python migrate.py for importing old JSON data into SQLite.

Is Docker supported?

Repo includes Dockerfile and docker-compose.yml. Compose mounts config.ini, data/, and logs/ from host.

What Python version is required?

Python 3.11 or newer.
